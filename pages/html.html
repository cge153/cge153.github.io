HTML stands for Hypertext Markup Language. It contains the text contents and references to other contents (e.g. images) of a web page and it also contains the code that tells the Browser how to display the web page.

A web page gets it's structre through HTML, which can also be seen as the sceleton of a web page.

The Hypertext part of HTML means that HTML documents can be linked and it is possible to navigate between HTML documents using these links. An HTML document is therefore not just a single, separate document, but can be part of a much bigger structure.

The Markup part of HTML is a reference to the code that is used to tell the browser how the contents of the HTML page should be displayed. This code - the HTML tags - enclose the actual contents and tell the browser to display text in e.g. bold.

The Basics

Semantic HTML

Semantic HTML elements should be prefered over non-semantic elements whenever possible. Why, you might ask?

  • It makes webpages more accessible (e.g. for screenreaders)
  • The webpage's SEO (Search Engine Optimazation) is improved
  • Reading the source code is much easier

Here are some examples of semantic HTML tags:

  • <header>
  • <nav>
  • <main>
  • <footer>
  • <article>
  • <section>
  • <aside>
  • <figure>
  • <figcaption>
  • <audio>
  • <video>

A Basic HTML File

The basic structure of a HTML file looks as follows:

Doctype

At the very first line of an HTML document, the DOCTYPE declaration tells the browser that the document is an HTML document.

Comments

To increase the readability of the HTML document, it can be very useful to add comments. This might also be helpful when testing code.
